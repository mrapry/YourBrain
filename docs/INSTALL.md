# Installation & Running YourBrain

YourBrain builds from source into a single binary (`yb` / `yb.exe`). This guide
covers **Windows**, **macOS**, and **Linux**, plus the optional ONNX embedder.

## Prerequisites

| Requirement | Why | Notes |
|---|---|---|
| **Rust toolchain 1.75+** | Compiles the project | Install via [rustup](https://rustup.rs). |
| **A C compiler** | Bundled SQLite compiles from source | MSVC on Windows, Clang/GCC on macOS/Linux. |
| **Git** | Clone the repository | Optional if you download a source archive. |

Check your Rust version:

```bash
rustc --version   # must be >= 1.75.0
cargo --version
```

---

## Windows

1. **Install the MSVC C++ build tools.** Install
   [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/)
   and select the **"Desktop development with C++"** workload (provides the MSVC
   compiler and Windows SDK needed to build bundled SQLite).

2. **Install Rust** with [rustup](https://rustup.rs) — accept the default
   `x86_64-pc-windows-msvc` host.

3. **Build** (PowerShell):

   ```powershell
   git clone <repo-url> yourbrain
   cd yourbrain
   cargo build --release
   # binary at .\target\release\yb.exe
   ```

4. **Run it:**

   ```powershell
   .\target\release\yb.exe stats
   ```

> If `cargo` is not recognized after installing rustup, open a new terminal, or
> add `%USERPROFILE%\.cargo\bin` to your `PATH`.

---

## macOS

1. **Install the Command Line Tools** (provides Clang):

   ```bash
   xcode-select --install
   ```

2. **Install Rust:**

   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   source "$HOME/.cargo/env"
   ```

3. **Build & run:**

   ```bash
   git clone <repo-url> yourbrain && cd yourbrain
   cargo build --release
   ./target/release/yb stats
   ```

Works on both Apple Silicon (`aarch64-apple-darwin`) and Intel
(`x86_64-apple-darwin`).

## Linux

1. **Install a C toolchain** (for bundled SQLite):

   ```bash
   # Debian / Ubuntu
   sudo apt-get update && sudo apt-get install -y build-essential pkg-config git

   # Fedora / RHEL
   sudo dnf install -y gcc gcc-c++ make pkgconf git

   # Arch
   sudo pacman -S --needed base-devel git
   ```

2. **Install Rust:**

   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   source "$HOME/.cargo/env"
   ```

3. **Build & run:**

   ```bash
   git clone <repo-url> yourbrain && cd yourbrain
   cargo build --release
   ./target/release/yb stats
   ```

## Put `yb` on your PATH

So you can type `yb` from anywhere (and so IDEs can find it), install the binary
into Cargo's bin directory (`~/.cargo/bin`, already on your PATH after rustup):

```bash
cargo install --path crates/yb                 # default (hash) embedder
cargo install --path crates/yb --features onnx # with the ONNX backend
```

Alternatively, copy `target/release/yb` (or `yb.exe`) to any directory on your
`PATH`.

> IDE MCP configs generated by `yb install` point at the **absolute path** of the
> binary that produced them, so they keep working regardless of your `PATH`.

## Verify the installation

```bash
yb config show     # prints version, config, and data/DB paths
yb stats           # prints memory counts and the active embedding model
```

Run the test suite to confirm a healthy build:

```bash
cargo test --all
cargo clippy --all-targets -- -D warnings
```

## Data directory & configuration

YourBrain stores everything locally. The data directory is resolved in this
order:

1. `YB_DATA_DIR` environment variable (overrides everything), else
2. Windows: `%APPDATA%\yourbrain`, otherwise `~/.yourbrain`.

It contains `config.toml`, the global `brain.db` (SQLite) and `brain.ybv`
(vector index), and one `dbs/<name>/` subfolder per isolated `--db-memory`
project. Copy [`config.example.toml`](../config.example.toml) to
`<data_dir>/config.toml` to customize defaults, or run `yb config show` to see
the resolved paths.

## Optional: ONNX embedder

The default embedder (`hash-bow-v1`) is dependency-free but only matches on
shared vocabulary — paraphrases with no common words score near zero. For true
semantic understanding, build the optional ONNX
[sentence-transformer](https://www.sbert.net/) backend, powered by
[`fastembed`](https://github.com/anush008/fastembed-rs) (tokenizer + pooling +
L2 normalization included). It is gated behind the `onnx` cargo feature so
default builds stay lean and portable.

### Build with the feature

```bash
cargo build --release --features onnx
```

> **Windows build note.** The release profile uses aggressive `lto`
> (`Cargo.toml` sets `lto = "thin"`), which can crash the `rustc` code generator
> (`STATUS_ACCESS_VIOLATION`, exit `0xc0000005`) while compiling the large ONNX
> dependency graph (`ort` / `onnxruntime`). If that happens, disable LTO for the
> ONNX build — the binary stays optimized:
>
> ```powershell
> # PowerShell (Windows)
> $env:CARGO_PROFILE_RELEASE_LTO="off"
> $env:CARGO_PROFILE_RELEASE_CODEGEN_UNITS="16"
> cargo build --release --features onnx
> ```
>
> ```bash
> # bash (Linux/macOS)
> CARGO_PROFILE_RELEASE_LTO=off CARGO_PROFILE_RELEASE_CODEGEN_UNITS=16 \
>   cargo build --release --features onnx
> ```

### Supported models

The model downloads from HuggingFace on first use and is cached on disk.

| Model key | Dim | Notes |
|---|---|---|
| `multilingual-e5-small` | 384 | Multilingual (great for mixed EN/ID); default choice. |
| `multilingual-e5-base` | 768 | Larger multilingual. |
| `all-minilm-l6-v2` | 384 | Fast English. |
| `bge-small-en-v1.5` | 384 | Strong English retrieval. |
| `paraphrase-multilingual-minilm-l12-v2` | 384 | Multilingual paraphrase. |

E5 models automatically get the `query:` / `passage:` instruction prefixes.

### Migrate an existing database

The vector dimension is locked when a database is created (ADR-5), so switching
models requires re-embedding. **Stop any running MCP server first**, then:

```bash
# Dry-run preview (omit --yes); add --yes to actually migrate:
yb reindex --provider onnx --model multilingual-e5-small --yes
```

`reindex` re-embeds every memory, rebuilds the vector index, clears the semantic
cache (its query vectors belong to the old model's space), and moves the ADR-5
lock to the new model.

To make ONNX the global default, set it in `config.toml`:

```toml
[embedding]
provider = "onnx"
model = "multilingual-e5-small"
# cache_dir = "~/.yourbrain/models"   # optional model cache location
```

After reindexing, CLI commands that touch that database need matching embedder
flags (global options on every subcommand):

```bash
yb --embedder onnx --embed-model multilingual-e5-small stats
```

For pointing an IDE's MCP server at ONNX, see
[docs/IDE_SETUP.md](IDE_SETUP.md#per-project-settings).

### Test the ONNX backend

`cargo test --all` uses the deterministic hash embedder (no network). Verify
ONNX manually with a **paraphrase recall** that shares no vocabulary with the
stored memory — something the hash embedder cannot match:

```bash
export YB_DATA_DIR=/tmp/yb-onnx-test   # PowerShell: $env:YB_DATA_DIR="$env:TEMP\yb-onnx-test"

yb remember "Backend uses Rust with the Axum web framework"
yb recall  "what language and library power the server"      # hash: weak / miss

yb reindex --provider onnx --model multilingual-e5-small --yes
yb --embedder onnx --embed-model multilingual-e5-small \
   recall "what language and library power the server"       # onnx: strong match
```

## Updating

```bash
cd yourbrain
git pull
cargo build --release
# if you used `cargo install`, re-run it to refresh ~/.cargo/bin:
cargo install --path crates/yb            # add --features onnx if you use ONNX
```

Your data in the data directory is preserved across rebuilds. Reload the MCP
server in your IDE after updating the binary.

---

## Uninstall

```bash
cargo uninstall yb          # if installed via `cargo install`
```

Then delete the build output (`target/`) and, if you want to remove all stored
memories, the data directory (`%APPDATA%\yourbrain` or `~/.yourbrain`, or your
`YB_DATA_DIR`).

## Troubleshooting

| Symptom | Fix |
|---|---|
| `cargo: command not found` | Open a new terminal or add `~/.cargo/bin` (`%USERPROFILE%\.cargo\bin`) to `PATH`. |
| Linker / `cc` / MSVC errors when building | Install the C toolchain (see per-OS prerequisites above). |
| `rustc` crash while building `--features onnx` on Windows | Build with `CARGO_PROFILE_RELEASE_LTO=off` (see the Windows build note). |
| `provider 'onnx' requested but built without the 'onnx' feature` | Rebuild with `cargo build --release --features onnx`. |
| `ModelMismatch` on a reindexed DB | Pass `--embedder onnx --embed-model <key>` (the DB was migrated to ONNX). |
| First ONNX command hangs | It is downloading the model from HuggingFace; needs internet on first run. |
